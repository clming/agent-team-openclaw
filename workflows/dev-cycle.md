# 🔄 开发循环工作流

## 流程概述

```
┌─────────────┐
│   新需求    │
└──────┬──────┘
       ▼
┌─────────────┐     ┌─────────────┐
│  产品经理   │────▶│   PRD 文档   │
│  需求分析   │     │             │
└─────────────┘     └──────┬──────┘
                           ▼
                    ┌─────────────┐     ┌─────────────┐
                    │  项目经理   │────▶│  任务清单   │
                    │  WBS 分解    │     │             │
                    └─────────────┘     └──────┬──────┘
                                               ▼
                                        ┌─────────────┐     ┌─────────────┐
                                        │  高级开发   │────▶│  代码实现   │
                                        │  编码实现   │     │             │
                                        └─────────────┘     └──────┬──────┘
                                                                   ▼
                                                            ┌─────────────┐     ┌─────────────┐
                                                            │   QA 测试   │────▶│  测试报告   │
                                                            │  测试验证   │     │             │
                                                            └─────────────┘     └──────┬──────┘
                                                                                       ▼
                                                                                ┌─────────────┐
                                                                                │  有 Bug?    │
                                                                                └──────┬──────┘
                                                                                   ┌───┴───┐
                                                                                  是      否
                                                                                   │       │
                                                                                   ▼       ▼
                                                                            ┌──────────┐  ┌──────────┐
                                                                            │ 返回开发 │  │  验收通过 │
                                                                            │  修复    │  │  上线    │
                                                                            └──────────┘  └──────────┘
```

## 各阶段详细说明

### 阶段 1：需求分析（产品经理）
**输入**: 用户原始需求
**输出**: PRD 文档
**验收标准**: 
- [ ] 用户故事完整
- [ ] 验收标准清晰
- [ ] 优先级已排序

### 阶段 2：任务分解（项目经理）
**输入**: PRD 文档
**输出**: WBS 任务清单
**验收标准**: 
- [ ] 任务粒度 ≤ 8 小时
- [ ] 依赖关系明确
- [ ] 里程碑设定

### 阶段 3：开发实现（高级开发）
**输入**: 任务清单
**输出**: 可运行代码
**验收标准**: 
- [ ] 代码通过自测
- [ ] 单元测试覆盖
- [ ] 文档完整

### 阶段 4：测试验证（QA 测试）
**输入**: 代码 + 测试用例
**输出**: 测试报告
**验收标准**: 
- [ ] 所有用例执行
- [ ] Bug 已记录
- [ ] 严重程度评估

### 阶段 5：Bug 修复循环
**条件**: 测试发现 Bug
**流程**: QA → 开发 → 修复 → 回归测试 → QA
**退出条件**: 无 Blocker/Critical 级别 Bug
